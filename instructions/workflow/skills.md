# Skills

- Keep skills project-agnostic and programming-language-agnostic by default.
- If narrower details are needed, prefer local instructions or separately loaded references instead of baking them into the skill body.
- Treat public skills installed from the Codex skills registry as externally managed.
- Do not edit registry-installed skills in place. Prefer project-local instructions, wrapper guidance, or a separate local skill instead.
- If skill provenance is unclear, call that out and avoid patching the skill without explicit user approval.
- Use `skills/INDEX.md` as the canonical inventory for skill provenance and ownership.
- Load `skills/INDEX.md` only when skill provenance, ownership, or maintenance scope is relevant to the current task.
